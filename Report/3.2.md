 Como a variável command_args[j] guarda o texto que o utilizador digitou no terminal, passar esta variável diretamente para o printf cria uma vulnerabilidade de String de Formatação (Format String Vulnerability). Se o utilizador digitar algo como %x %x %x, o programa vai ler e imprimir dados confidenciais diretamente da Stack.

 ```

```

 A variável i é um int (aceita números negativos), mas a constante N_AVAILABLE_COMMANDS (ou o tamanho do array) é do tipo unsigned long (apenas números positivos). Comparar variáveis com sinais diferentes (Signedness Mismatch) pode causar falhas de lógica ou loops infinitos se o valor sofrer um overflow.

 ```
int i;
for (i = 0; i < N_AVAILABLE_COMMANDS; i++) {
```

for (size_t i = 0; i < N_AVAILABLE_COMMANDS; i++) {

No commands.c a função gets é uma das funções mais perigosas do C porque lê tudo o que o utilizador digita sem verificar se cabe no espaço de memória alocado (Buffer Overflow). Isto permite fazer smash na Stack do programa e reescrever o endereço de retorno das funções.

Correcão, limitar o tamanho máximo de caracteres lidos.
```
fgets(buffer, sizeof(buffer), stdin);

### 3.2 Clang static analyser

- No core.c:207.

Na tentativa de remover um utilizador da lista ligada, o código liberta a memória do nó (node->next) e, na linha imediatamente a seguir, tenta ler o ponteiro node->next->next de dentro do bloco que acabou de ser destruído. Isto viola as regras de isolamento de memória. Se o gestor de memória reutilizar esse bloco libertado para outra variável, o programa lerá dados corrompidos, resultando num comportamento indefinido ou num crash imediato por falha de segmentação (Segmentation Fault).
